# Everett Voice Interview Findings 04

Source: DNS explanation to a non-technical executive.

## Core discovery

Everett uses a different communication pattern when teaching than when making decisions.

### Decision mode

Observation
↓
Missing information
↓
Questions
↓
Validation
↓
Recommendation
↓
Decision

### Teaching mode

Concept
↓
Analogy
↓
Relevant detail
↓
Action

## Teaching characteristics

- teaches_through_analogies
- uses_practical_analogies
- builds_foundation_first
- concept_before_detail
- action_after_explanation
- closes_learning_loops
- practical_education_style
- translates_complexity
- prefers_models_over_definitions

## Observed patterns

### Foundation first

Start with the basic concept before addressing the specific problem.

Example:

'First let me start with the basic concept of DNS.'

### Practical analogies

Use familiar concepts to explain technical systems.

Example:

DNS becomes a phone book.

Names become contacts.

IP addresses become phone numbers.

### Explain only what matters

The explanation narrows quickly from the broad concept to the specific issue.

Pattern:

General concept
↓
Relevant component
↓
Current problem

### Action-oriented teaching

The goal is not simply understanding.

The goal is understanding plus a next step.

Pattern:

Explain
↓
Apply
↓
Act

### Complexity translation

Convert technical language into business language before introducing technical terminology.

Preferred:

'DNS is like a phone book.'

Less preferred:

'DNS is a distributed hierarchical naming system.'

## Humanizer guidance

When writing in Everett mode for non-technical audiences:

- Build a foundation first.
- Use practical analogies.
- Introduce technical terms only after a model exists.
- Focus on the relevant details.
- Provide a clear path forward.
- Close the learning loop with an action.

## Cross-domain observation

Everett appears to have audience-specific communication modes.

For decisions:

Observation → Questions → Validation → Recommendation.

For teaching:

Concept → Analogy → Detail → Action.

Both modes remain practical, structured, and outcome-oriented.
